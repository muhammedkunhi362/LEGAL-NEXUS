# Product Requirements Document (PRD)

**Project Name:** Legal Nexus Advocates & Consultants Website
**Document Version:** 1.1
[cite_start]**Goal:** To design and develop a modern, highly professional, and strictly BCI-compliant web presence for the firm[cite: 1].

---

## 1. Technical Specifications
* **Framework:** Next.js for robust server-side rendering, routing, and optimal SEO performance.
* **Styling:** Tailwind CSS for rapid, utility-first UI development, ensuring precise execution of the specific color palette and UI design components.
* **Responsiveness:** The website must be fully responsive, seamlessly adapting to desktop, tablet, and mobile viewports.
* **Interactive UI Elements:** Implement 3D animations triggered by scrolling on the Home Page to create an engaging, highly modern user experience without compromising the solemn professional tone.

---

## 2. Design & UI Specifications
* **Color Palette:**
    * **Primary Background (80%):** Off-White (`#FFFAF2`) to create a clean, highly readable canvas.
    * **Primary Accent (15%):** Orange (`#EF713D`) used strategically for buttons, active states, and highlighting key elements.
    * **Secondary Accent (5%):** Light Blue/Gray (`#C3CFD9`) for subtle card backgrounds and hover states.
* **Typography:** Large, bold sans-serif fonts for headings with high legibility for body text.

---

## 3. Site Architecture & Content Requirements

* [cite_start]**Function:** A mandatory entry page where users must accept a disclaimer before entering the website[cite: 2, 3].
* [cite_start]**Purpose:** To confirm that the visitor is accessing the site voluntarily and that the content is only informational[cite: 4].
* **Key Points to Include:**
    * [cite_start]The website is only for informational purposes[cite: 6].

    * [cite_start]The information should not be treated as legal advice[cite: 8].

    * [cite_start]The user accessed the site at their own request[cite: 10].
